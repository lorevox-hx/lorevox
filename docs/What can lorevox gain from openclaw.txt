# OpenClaw‑Style Memory & RAG Patterns in LoreVox

This document maps OpenClaw’s behavioral patterns for **persistent memory** and **RAG** onto LoreVox, using your existing architecture and v7 direction. The goal is not to import OpenClaw’s code, but to **absorb its patterns** into a LoreVox‑native agent loop.

---

## 1. Mental model

- **LoreVox today:**  
  Local‑first memoir lab with rich schema (memory, snippets, timeline, embeddings, tags), LLM/TTS/Whisper stack, and RAG endpoints.

- **What we’re adding (from OpenClaw’s patterns):**
  - **Agent loop** around every message
  - **Memory hooks** before/after LLM and tools
  - **Memory policies** (what to store, update, merge, decay)
  - **RAG as a skill** (invoked by the agent, not the user)
  - **Reflection** (summarize + write back to memory each turn)

Think of it as layering a **cognitive process** on top of your existing APIs and schema.

---

## 2. High‑level pipeline

Proposed LoreVox agent loop (OpenClaw‑style):

1. **Receive** user message
2. **Before LLM hook**
   - Recall relevant memories
   - Optionally run RAG
   - Build system/context prompt
3. **LLM step**
   - Decide: respond directly, call tools, or both
4. **After LLM hook**
   - Parse tool calls
   - Decide what to store as “intent” or “plan”
5. **Tool execution**
   - Call skills (RAG, memory, TTS, etc.)
6. **After tool hook**
   - Reflect on results
   - Summarize
   - Write/update memory
   - Update timeline
7. **Return** final response to user

# server/agent_loop.py

from .hooks import before_llm, after_llm, after_tool
from .llm import run_llm
from .tools import execute_tools_if_any
from .reflection import run_reflection

def process_message(user_id: str, message: str) -> str:
    # 1. Before LLM: recall + optional RAG
    context = before_llm(user_id=user_id, message=message)

    # 2. LLM: decide response + tool calls
    llm_output = run_llm(user_id=user_id, message=message, context=context)

    # 3. After LLM: parse tool calls, update intent memory if needed
    tool_calls, updated_context = after_llm(
        user_id=user_id,
        message=message,
        llm_output=llm_output,
        context=context,
    )

    # 4. Execute tools (skills)
    tool_results = execute_tools_if_any(
        user_id=user_id,
        tool_calls=tool_calls,
        context=updated_context,
    )

    # 5. After tool: reflection + memory write
    final_response = after_tool(
        user_id=user_id,
        message=message,
        llm_output=llm_output,
        tool_results=tool_results,
        context=updated_context,
    )

    # 6. Optional: run deeper reflection (session summary, long-term memory)
    run_reflection(
        user_id=user_id,
        message=message,
        llm_output=llm_output,
        tool_results=tool_results,
        context=updated_context,
    )

    return final_response

from . import skills

_SKILL_MAP = {
    "rag_search": skills.skill_rag_search,
    "write_memory": skills.skill_write_memory,
    "query_memory": skills.skill_query_memory,
    "tts": skills.skill_tts,
    "whisper": skills.skill_whisper,
}

def execute_tools_if_any(user_id: str, tool_calls: list, context: dict) -> list:
    results = []
    for call in tool_calls:
        name = call["name"]
        args = call.get("arguments", {})
        fn = _SKILL_MAP.get(name)
        if not fn:
            continue
        result = fn(user_id=user_id, **args)
        results.append({"name": name, "result": result})
    return results

from .memory import create_memory_entry, update_memory_entry, find_similar_memories

def should_store_message(message: str, llm_output: dict) -> bool:
    """
    Example heuristics:
    - User shares a preference
    - User shares a biographical detail
    - User defines a project, goal, or plan
    - LLM marks content as 'memory-worthy' via a hidden tag
    """
    # Simple placeholder: later, use an LLM classifier or rules
    text = (message + " " + llm_output.get("text", "")).lower()
    keywords = ["i prefer", "my favorite", "i live in", "my goal", "i want to"]
    return any(k in text for k in keywords)


def store_message(user_id: str, message: str, llm_output: dict):
    """
    - Decide whether to create or update memory
    - Use memory_depth, tags_json, and embeddings

    # 1. Extract candidate memory snippet
    snippet = _extract_memory_snippet(message, llm_output)

        update_memory_entry(user_id=user_id, existing=similar[0], new_snippet=snippet)
    else:
        # 4. Create new memory entry
        create_memory_entry(user_id=user_id, snippet=snippet)

from .db import get_db
from .embeddings import embed_text

def recall_relevant_memories(user_id: str, query: str, limit: int = 8) -> list:
    """
    - Use embeddings + tags_json + memory_depth
    - Return snippets to inject into the LLM context
    """
    db = get_db()
    query_vec = embed_text(query)

    # Pseudocode: vector search + filters
    rows = db.search_memories(
        user_id=user_id,
        embedding=query_vec,
        limit=limit,
        min_depth=1,
    )
    return [row.to_dict() for row in rows]


def create_memory_entry(user_id: str, snippet: dict):
    db = get_db()
    embedding = embed_text(snippet["text"])
    db.insert_memory(
        user_id=user_id,
        text=snippet["text"],
        tags_json=snippet.get("tags", {}),
        memory_depth=snippet.get("depth", 1),
        embedding=embedding,
    )


def update_memory_entry(user_id: str, existing: dict, new_snippet: dict):
    db = get_db()
    # Example: merge text or append note, bump depth
    merged_text = existing["text"] + "\n\n" + new_snippet["text"]
    new_depth = max(existing["memory_depth"], new_snippet.get("depth", 1)) + 1

    db.update_memory(
        id=existing["id"],
        text=merged_text,
        tags_json=_merge_tags(existing["tags_json"], new_snippet.get("tags", {})),
        memory_depth=new_depth,
    )


def find_similar_memories(user_id: str, text: str, limit: int = 3) -> list:
    db = get_db()
    vec = embed_text(text)
    rows = db.search_memories(user_id=user_id, embedding=vec, limit=limit)
    return [r.to_dict() for r in rows]

    - Write to memory and timeline
